# 產品需求文件 (PRD)

## 專案名稱
食譜收藏夾系統

## 專案描述
這是一個提供使用者收藏、管理食譜，並能根據手邊食材搜尋合適料理的網站服務。

---

## 1. 專案概述

### 背景與動機
現代人常在網路上尋找食譜，但這些食譜散落各處難以集中管理；此外，使用者常常遇到「打開冰箱有特定食材，卻不知道能煮什麼」的痛點。因此，開發一個能集中收藏食譜，並支援使用「食材組合」來反向搜尋食譜的系統，能有效解決這些日常烹飪的情境難題。

### 目標用戶
- **一般人 (一般使用者)**：喜歡自己下廚，需要收集與管理食譜，或經常需要依據剩餘食材找靈感的人。
- **管理員**：負責維護系統、審核內容與管理平台社群秩序的人員。

### 核心價值主張
提供一個直覺、便利的集中式食譜管理平台，並運用食材組合搜尋功能，幫助使用者最大化利用現有食材、解決每天「要煮什麼」的煩惱。

---

## 2. 功能需求

以下列出系統的 5 項主要功能與對應的使用者故事 (User Story)：

1. **儲存/收藏食譜**
   - 作為 **一般人**，我希望 **能夠將感興趣的食譜加入我的收藏夾**，以便 **我未來準備做菜時能快速找到並參考**。
2. **瀏覽與搜尋食譜**
   - 作為 **一般人**，我希望 **能瀏覽系統推薦食譜並使用關鍵字搜尋**，以便 **根據我想吃的料理名稱或菜系快速找到作法**。
3. **從食材組合搜尋食譜**
   - 作為 **一般人**，我希望 **能輸入手邊現有的多種食材組合來進行搜尋**，以便 **找到能利用這些材料製作出來的料理，避免食材浪費**。
4. **會員註冊與登入管理**
   - 作為 **一般人**，我希望 **建立個人帳號並登入系統**，以便 **妥善保存我的專屬食譜收藏夾，並在不同設備上同步**。
5. **後台內容與用戶管理**
   - 作為 **管理員**，我希望 **能瀏覽、編輯、刪除全站食譜內容與管理用戶狀態**，以便 **維持平台的內容品質與良好的社群環境**。

---

## 3. 非功能需求

### 技術限制
- **後端框架**：使用 Python Flask 建立 Web 伺服器與 API 路由。
- **前端頁面**：使用 Jinja2 模板引擎進行頁面渲染，搭配 HTML/CSS/JS (不依賴大型前端框架)。
- **資料庫**：使用 SQLite 作為關聯式系統資料庫。

### 效能與安全考量
- **資訊安全**：使用者密碼必須採用雜湊加密 (如 bcrypt) 儲存；必須過濾使用者輸入內容避免 XSS 攻擊；操作資料庫時需使用 ORM (如 SQLAlchemy) 或參數化查詢防護 SQL Injection。
- **查詢效能**：「食材組合搜尋」涉及多對多或模糊比對，需注意資料庫查詢 (Query) 最佳化或建立索引，避免造成資料庫過載影響回覆速度。

## 4. MVP (最小可行性產品) 範圍

為了快速驗證市場假說，系統的第一階段上線範疇優先級如下：

- **Must Have (必須有 - 核心功能)**
  - 基礎會員註冊與登入
  - 瀏覽食譜列表、食譜詳細資訊頁面
  - 關鍵字搜尋食譜
  - 使用者的食譜收藏 (加入收藏/取消收藏/檢視收藏清單)
- **Should Have (應該有 - 重要價值)**
  - 最多輸入 3 項食材的「食材組合搜尋」功能
  - 基本的管理員後臺介面 (能刪除不當食譜)
- **Nice to Have (加分項 - 增強體驗)**
  - 使用者自行上傳建立新食譜分享
  - 食譜分類標籤 (如：日式、地中海、素食)
  - 點讚與評論功能
